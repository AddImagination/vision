## 스터디 논의 필요 사항

1. 주제 선정 : 비전
    - 논문 선정 기준
        - 시간 순으로 살펴 봄. 
        - 핵심이 되는 내용부터 본 후 세부적인 내용을 추가해서 성공한 사례를 확인. (e.g.GAN → DCGAN, RNN → LSTM)
        - 논문 내용을 바탕으로 구현된 코드가 존재해야 함.               

2. 협업 방식 논의 
    - 도구 : Github
    - 작성 규칙 
        - 업로드 규칙(git) : 어떤 방식으로 git에 수정한 자료를 업로드할 지에 대한 규칙 논의 필요. (git 학습 필요.)
        - 작성자 규칙(ref) : 공동 작업 시 문서 내 본인이 찾은 내용에 대한 상세한 출처 작성.
        
3. 발표 내용 세분화/배분
    -  1인이 미리 논문의 내용을 훑어보고 각자 파트에 대한 배분을 통해 상세 내용의 경우 맡은 부분이 겹치지 않게 조정
    논문 리뷰 시 동일인이 최초 개략적인 흐름 및 내용 리뷰 진행 후 상세 내용에 대한 공유 진행.

4. 발표 방식
    - 자료 : 전 주에 학습한 논문 내용 중 본인이 맡은 내용을 참고 자료를 바탕으로 정리하여 발표.
    - 순서 : 논문의 이해를 위해 흐름에 맞게 순서대로 내용이 나올 때마다 발표 진행. 
        
5. 교육 자료 구상
    - 정리 방안 
        - tree 형식으로 나타내어 해당 분야에 대해서 알기 위해 학습해야 할 내용을 확인 함.
        - 논문의 흐름대로 나열하여 시간 순으로 어떤 식으로 연구되었는 지 확인 함. 
        - 블로그 업로드 
       
6. 규칙 
    - 벌금 : 지각(5,000원), 결석(10,000원)
    - 일시, 시간 : 매주 일요일 오전 10시 - 12시, 강남역 
    - 시작 시간 : 10시 5분

위 내용은 첫 모임 시 정한 주제와 세부 규칙 사항들입니다. 작성 규칙이나 블로그 운영 등 더 상세하게 
정할 필요가 있는 부분에 대해서는 1달간 시행착오를 겪으며 하나씩 추가해나갈 예정입니다.
